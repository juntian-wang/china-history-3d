# 2026-07-06 工作日志

## 修复 globe 地球仪球体贴图不显示
- 问题：用户反馈 `china_map_globe.html` 中地球仪为黑色，朝代着色的中国地图没有贴在球面上
- 根因：`globe.baseTexture` 被错误地传入了 echarts 实例对象，而 echarts-gl 期望 HTMLCanvasElement / Image / URL 字符串
- 修复：
  - 新增 `textureCanvas` 变量保存 offscreen echarts 实例的 viewport canvas
  - 拆分 `paintTexture()`（仅更新 offscreen canvas）与 `commitTexture()`（复制 canvas 并触发 globe 重新上传）
  - 初始 `baseTexture` 改为 `textureCanvas`
  - 朝代切换动画中每帧只 `paintTexture()`，动画结束后再 `commitTexture()`，避免每帧创建新 canvas 造成卡顿
  - 主题切换、省界开关等即时操作仍调用 `updateTexture()` 直接刷新球面
- 验证：本地 http.server 3000 端口可正常服务修改后的文件（需注意环境 HTTP 代理，使用 no_proxy 或 --noproxy localhost）

## 修复 globe baseTexture "e.match is not a function" 错误
- 第一版修复（传 canvas 对象）失败：echarts setOption 合并 option 后，canvas 对象的 `instanceof HTMLCanvasElement` 检查失效，echarts-gl 的 `Ha()` 函数返回 false，导致走到 `e.match()` 分支报错
- echarts-gl 源码分析：`loadTexture` 先检查 `Ha(e)`（HTMLCanvasElement/HTMLImageElement/Image），为 true 则创建 Texture2D；否则当作字符串 URL 处理，调用 `e.match()`
- 根因：echarts 5 的 option 合并机制会破坏 canvas 对象引用
- 最终修复：baseTexture 改用 `canvas.toDataURL('image/png')` 生成 dataURL 字符串，echarts-gl 走字符串 URL 加载路径
- 性能权衡：2048×1024 PNG dataURL 约 500KB-1MB，朝代切换动画中只更新 offscreen canvas，动画结束后 commitTexture 一次 toDataURL，避免每帧卡顿
