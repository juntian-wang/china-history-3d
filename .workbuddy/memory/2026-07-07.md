## 修复：$('loader p') 返回 null 导致页面完全无法加载 (00:18)
- **根因**：`$` 函数定义为 `id => document.getElementById(id)`，但代码中多处使用 `$('loader p')` 和 `$('loader .ring')`，这些是 CSS 选择器而非合法 ID，getElementById 返回 null
- 一旦执行 `null.textContent = '...'` 就抛出 TypeError，init() 中断，页面永远卡在"正在加载"
- 此 bug 一直存在但之前未触发——因为只在错误处理路径中使用，正常加载时不会走到
- 新增的加载状态提示（`_loaderP.textContent = '正在加载历史数据…'`）在正常路径中调用了它，暴露了 bug
- **修复**：init() 中 `$('loader p')` → `document.querySelector('#loader p')`，onMapLoad catch 块同理
- 验证：用户确认"可以了"，页面正常加载
