# 中国历史疆域 3D 地形图

基于 MapLibre GL JS 的交互式历史地图，在真实卫星影像和 DEM 高程 3D 地形上，可视化展示中国从春秋时期（公元前 770 年）到现代（1949 年）的疆域变迁。内容以部编版《中国历史》七年级上下册和八年级上册教科书为蓝本。

---

## 功能特性

- **60 个历史时间断面**，覆盖从春秋到现代约 2700 年的疆域变迁
- **24 个朝代快捷按钮**，一键跳转到对应朝代的有效断面
- **真实 3D 地形**：基于 AWS Terrarium DEM 高程数据，3D 地形起伏
- **卫星影像底图**：Esri ArcGIS World Imagery 卫星影像
- **时间轴**：拖拽滑块逐断面查看，支持播放/暂停（2.5s/断面）
- **键盘快捷键**：空格播放/暂停、左右箭头切换断面、ESC 暂停
- **720ms 平滑过渡动画**：朝代切换时省份颜色渐变
- **15 条历史河流**（蓝色发光线）+ **21 座山脉** + **3 处沙漠** + **22 座历史名城**（点击查看详细描述）
- **点击省份**：弹出信息卡片，显示政权、年代、都城、辖地和描述
- **108 个历史事件**（覆盖七年级上/下 + 八年级上册），按年份自动匹配到对应断面
- **事件详情卡片**：点击事件标记查看描述、时间、地点，配合 AI 生成版画插图
- **108 张 AI 历史插图**：晚明徽州木刻版画风格，黑白白描线描
- **2D/3D 视角切换**：右上角按钮，一键在俯视和 3D 立体视角间切换
- **点击事件自动切 3D**：点击历史事件时自动切换到 3D 立体视角
- **时间轴归位**：点击时间线或朝代按钮时，自动飞回默认视角
- **古地图风格 UI**：暗色暖调（古铜色 `#c8a050`）、楷体竖排标注、柔光云雾飘动效果、开场云雾散开动画
- **归位按钮**：一键飞回中国俯瞰视角
- **双模式运行**：支持 HTTP 服务器部署，也支持直接双击 HTML 文件打开（数据已内嵌）
- **移动端适配**：响应式布局，小屏设备可用

## 数据管道

### 1. 事件数据（108 个历史事件）

1. 从部编版初中历史教科书（七年级上/下、八年级上）中挑选 **108 个关键历史事件**
2. 每个事件包含：事件 ID（教材编号体系）、事件名称、公历年份、朝代、时期、地点、省份、经纬度、详细描述
3. 数据最终以 JavaScript 数组 `EVENTS_RAW` 形式内嵌在 `maplibre_3d_history.html` 中

> **教材编号说明**：`7A` = 七年级上册（古代史），`7B` = 七年级下册（古代史续），`8A` = 八年级上册（近现代史）。编号格式 `{教材}-{单元}-{序号}`，如 `7A-02-10` 表示七年级上册第 2 单元第 10 条。

### 2. 事件图像生成（108 张 AI 插画）

1. 从事件数据中读取每个事件的提示词
2. 每个提示词附加统一风格后缀：晚明徽州木刻版画风格，黑白白描线描
3. 通过 AI 图像生成模型生成 768×1152 的 PNG 图片
4. 文件命名：`{教材编号}_{事件名}.png`，存放在 `event_images/` 目录
5. 在 HTML 中通过 `EVENT_IMAGE_MAP` 查找表将事件 ID 映射到对应图片文件名

### 3. 事件图像压缩优化（2026-07-08）

1. 原始 108 张 PNG（每张 ~2MB，总计 **225MB**）在弹出卡片中仅以 140×210px 显示，严重浪费
2. 使用 Pillow 库批量处理：
   - 缩放至 280×420px（2x 显示尺寸，兼容 Retina）
   - 转为 WebP 格式（quality=80）
3. 原始 PNG 全部备份至 `backup/` 目录（228MB），可随时回退
4. HTML 中图片引用从 `.png` 改为 `.webp`
5. **压缩效果**：单张 ~2MB → ~32KB，总大小 **225MB → 3.4MB（缩减 98.5%）**，加载速度提升约 30 倍

### 4. 疆域/政权数据（60 个历史断面）

`territories_merged.json` 是项目的核心数据文件，构建流程：

1. 以现代中国 34 个省级行政区为近似单元
2. 按每个历史断面，将各省份分配给当时的各个政权
3. 每个政权标注：名称、辖地省份、都城名+坐标、时间范围、教科书描述、调色板颜色
4. 数据经过多次丰富化：不仅包含名称和省份，还包含 `desc`（教科书描述）、`years`（起止年代）等字段

## 本地运行

### 方式一：直接双击打开（推荐）

`maplibre_3d_history.html` 已将所有疆域和底图数据内嵌，**无需 HTTP 服务器**：

1. 在 Finder 中双击 `maplibre_3d_history.html`
2. 浏览器会自动打开，地图即可正常显示
3. 事件图片（`event_images/` 目录）需要和 HTML 在同一目录下

### 方式二：Python HTTP Server

```bash
cd /path/to/project
python3 -m http.server 8780
```

浏览器访问：**http://localhost:8780/maplibre_3d_history.html**

### 方式三：其他静态服务器

```bash
# Node.js
npx serve -p 8780

# PHP
php -S localhost:8780
```

## Docker 部署

```bash
cd /path/to/project

# 构建并启动
docker compose up -d --build

# 访问 http://localhost/

# 停止
docker compose down
```

Docker 镜像基于 `nginx:alpine`，将 HTML、event_images 和 website 目录复制到容器内，包含健康检查和自动重启策略。

### 部署到云服务器

1. 购买轻量应用服务器（腾讯云、阿里云等），选择中国香港 / 新加坡节点可免备案
2. 安全组/防火墙放行 80 端口
3. 安装 Docker 和 Docker Compose
4. 上传整个项目目录到服务器
5. 执行 `docker compose up -d --build`
6. 通过 `http://<服务器IP>/` 访问

## 已知限制与注意事项

- **疆域为近似值**：历史疆域以现代省份边界近似划分，不代表严格的历史边界线。数据仅供教学和可视化示意参考。
- **AI 生成图像**：所有 108 张事件插图由 AI 模型生成，风格统一但非史实场景的精确复原。部分图像可能包含不准确的历史细节。线上版本已转为 WebP 并缩放。
- **事件匹配**：108 个事件与 60 个断面按年份最近原则自动关联。部分断面可能没有对应事件，部分断面可能关联多个事件。
- **DEM 精度**：地形高程数据来自 AWS Terrarium 项目（最大 zoom 15），在最大缩放级别下可能呈现阶梯状。
- **首次加载**：Esri 卫星瓦片和 DEM 瓦片需从网络加载，初次加载时卫星底图可能逐块显示。事件插画已压缩为 WebP（总大小仅 3.4MB），加载极快。

## 项目制作过程

本项目从零构建，大致经历以下阶段：

1. **数据整理**：从初中历史教科书中提炼 108 个历史事件，整理为结构化数据（事件名称、年份、朝代、地点、经纬度、描述）
2. **疆域建模**：构建 60 个历史断面的政权-省份映射关系，基于现代 34 个省级行政区的近似疆域表达
3. **地图搭建**：使用 MapLibre GL JS 构建单文件 HTML 网站，集成卫星底图、3D 地形、多图层控制
4. **时间轴系统**：实现断面切换、朝代快捷跳转、自动播放、颜色过渡动画
5. **地理标注**：添加河流、山脉、沙漠、历史名城等地理要素图层
6. **事件系统**：将 108 个历史事件嵌入 HTML，实现地图标记、侧边列表、详情卡片
7. **AI 图像生成**：为每个历史事件生成版画风格插画（108 张），集成到事件详情和独立画廊
8. **数据丰富化**：为所有政权和事件添加教科书级详细描述文本，补充 8A 近代史事件
9. **UI 打磨**：古地图风格主题、响应式布局、键盘快捷键、移动端适配、标题文字阴影
10. **视角交互**：2D/3D 视角切换按钮、点击事件自动切 3D、时间轴归位动效
11. **离线化**：数据内嵌至 HTML、MapLibre 本地化，支持双击直接打开
12. **图片压缩**：将 108 张 AI 插画缩放为 280×420px 并转 WebP，总大小从 225MB 降至 3.4MB（缩减 98.5%）
